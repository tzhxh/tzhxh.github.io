关于函数传递参数的一点疑问

发表于 2021-03-23

在日常写代码的时候，我们不可能将所有的逻辑，所有的代码都写在一个main函数中。我们往往会将很多代码逻辑写在不同的函数中以便调用，往往这些函数在定义时会写上形参。然而当我们的理解不到位时就会变得混乱 我们先来看几个例子： 例 1： 

public class Main {
public static void main(String[] args) {
    int i = 0;
    change(i);
    System.out.println(i);
}

public static void change(int j){
    j = 1;
}

}

这段代码会输出什么呢？ 0 还是 1？ 答案是0。

那么这个呢

例 2：

public class Main { public static void main(String[] args) { int[] s = {0, 1}; change(s); System.out.println(s[0]); }

public static void change(int[] temp){
    temp[0] = 2;
}

}

这个的结果又会是什么呢？ 0还是2？ 答案是2。

例 3：

public class Main { public static void main(String[] args) { int[] s = {0, 1}; change(s); System.out.println(s[0]); }

public static void change(int[] temp){
    temp = new int[]{2, 1};
}

}

这个的结果0，这是为什么呢？

那么再看 例 4： 

public class Main {
public static void main(String[] args) {
    String s = "1";
    change(s);
    System.out.println(s);
}

public static void change(String temp){
    temp = "2";
}

}

这一个的结果又是什么？ 答案是1。

从上面的例子我们可以看出有一些change方法并没有影响到原来的值，这是为什么？

众所周知，Java的数据类型可以分为两类基本数据类型和引用数据类型(其实很多语言貌似都可以这样分)，像 int, float...之类的是基本数据类型(我认为是那几个以小写字母开头的几个类型)。 而产生区别的也就是两大类型的参数传递之间的差异。我们知道形参只有在调用这个函数时传入了实参才能发挥作用。在之前，我一直认为在传入实参时我传入的是什么，用的就是什么，因此走进了一个误区。 我在查阅资料的时候，发现有些人说是传递的实参是你传递的变量的拷贝，所以基本类型的不会受到影响，而引用类型的虽然也是拷贝，但实际上传的是地址，所以直接修改值会受到影响。 但为了自己好记，我用这种方式来理解，形参实际上是在调用这个方法时创建的一个变量，是一个独立的变量，我们在将实参传入时，其实是将实参赋值给形参的一个过程，那么既然是单独的变量，那就有属于自己的内存。因此，对于基本类型来说， 我们将基本类型传入时，其实是基本类型的赋值，形参拿到了实参的值，但两个是独立的变量，互不影响。对于引用类型来说，也一样是赋值，但引用类型的赋值给的是地址，也就是说形参和实参两个单独的变量都存了同一个地址指向同一个对象，因此例子2和例子3会出先不同的现象也就能够理解，注意例2是对他们所指向的对象进行修改。而例3则是将形参所指向的地址改变了，那么形参和实参指向的就不是同一个地址了，也就不是同一个对象了。至于例4，和例3 是一样的。因为String这个类型特殊一点， s = "1"这个代码就把它当成 String s = new String("1")来理解吧。具体的就先不深究了，后面有时间再看看吧。。。。
