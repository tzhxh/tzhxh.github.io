git使用小结

发表于 2022-01-06

从使用场景和常用命令来记录一下git的使用

场景分析：

场景一： 直接克隆项目

git clone 命令

场景二： 远程有一个空仓库，本地有一个项目，将本地项目推上远程仓库

  1. git init 初始化项目为git项目

  2. git remote add origin + 远程仓库地址 链接到远程仓库

  3. git add ./ 将代码添加到暂存区

  4. git commit -m “提交的代码的一些说明” 提交暂存区的代码到本地

  5. git push origin/对应分支 推到远程仓库

  6. 使用git push --set-upstream origin/对应分支 推到远程仓库并且后续使用push可以不用输入origin/对应分支

场景三： 本地有一个分支，但远程仓库没有分支将本地分支推到远程仓库并建立对应分支
  1. git push orogin/对应分支
场景四： 基于以存在的分指创建一个新的本地分支，并推上远程仓库

  1. git checkout -b 要创建的分支名 被基于的分支名 说明： 被基于的分支名可以不填，那就是基于当前的本地分支， 也可以填origin/分支名，那就是基于远程分支

  2. git push origin/对应分支

场景五： 提交了错误代码并推到了远程，需要撤回

  1. git reset --soft/hard HEAD^ 这个操作可以撤销一步 说明： 参数使用soft的话，撤销提交后会保留代码，使用hard的话，撤销提交后代码不会被保留。

  2. git push --force 强制提交到远程仓库 说明： git reset --soft/hard 后面可以跟参数撤销某一次指定的提交 参数可以通过git log来查看提交的日志

场景六： 本地代码修改过后需要提交到远程

  1. git pull 拉下最新代码

  2. git checkout 切换到自己的分支

  3. git merge master 合并主分支代码

  4. git add./ 提交代码到暂存区

  5. git commit -m ‘’ 提到暂存区代码到本地

  6. git push 推到远端

场景七 在错误的分支写代码

  1. git stash save ‘本次存储的说明或者名称’ 将代码新增的代码存起来

  2. git checkout 切换到正确的分支

  3. git stash pop 将存储的代码推出栈顶的那一个

说明： git shash pop 可以输入参数推出指定的某一次存储的代码

常用命令解析

git clone

从远程分支克隆代码下来到本地

git init

将本地项目初始化为可以使用git进行管理的项目

git add

将代码添加到暂存区

git commit

将暂存区的代码提交到本地提交

git push

将本地的提交推到远端 ，当后面有远程分支名时，表示push到指定分支，若远程仓库不存在该分支，则会自动创建，具体命令为 git push origin/分支名。还可以使用参数 --set-upstream与远程仓库的对应分支进行关联，后续直接使用git push就可以推到该分支不必每次输入后面的远程分支名。具体命令为 git push --set-upstream origin/分支名

git branch

查看分支，有参数r 表示查看远程分支，具体命令为 git branch -r。还有参数a表示查看全部分支

git branch newBranch orogin/branchName 表示创建一个新的本地分支并关联远程已经存在的分支

git checkout 分支名

切换分支，有参数b表示基于某一分之创建新分支，具体命令为 git checkout -b newBranch [branchName]

git pull

从远程分支拉取最新代码 并且直接merge

git fetch

从远端拉取最新代码，但不merge，需要自己使用merge命令 说明： 还有其他区别，但我不太清楚

git reset --soft/hard 某一次提交

可以撤销某一次提交， 如果是HEAD^，表示提交最近的一次提交，具体命令为git reset --soft/hard HEAD，如果是HEAD^，可以表示撤销最近的两次提交。

此外，可以通过git log 命令查看提交记录，并看到每次提交的id，通过这个来撤销指定的某一次提交，具体操作我得再看看。。。

此外，soft/hard 是表示两种不同的模式，如果是soft则撤销后代码会保留，当成新的修改。如果是hard则撤销后代码消失，不被保留。

git stash

存储代码或者将代码从存储中取出

使用git stash save ‘label’ 就是将修改的代码存储起来

使用git stash pop 就是将存储起来的代码取出来，不加任何的参数就说取出栈顶的那个，

可以使用git stash list 查看存储的列表

还有个git stash apply 命令，具体使用还不清楚

git status

查看当前的git状态

git log

查看git提交日志

说明： []中的参数表示可有可无
